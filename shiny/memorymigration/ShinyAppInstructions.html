Memory Migration Shiny App Interactive Tool Instructions

On the left side of the screen labeled as Population:

1.     Select the initial distribution of the population in year 0 as either:

a.     Optimal – The initial distribution of the population will match the first year of the resource. The example below depicts an optimal initial distribution of the population to match a resource with

 

b.     Non – migratory – The initial distribution of the population has no starting migratory behavior

 

c.     Sinusoidal – The initial distribution of the population will be a sinusoidal wave. The example below depicts an optimal initial distribution of the population with

 

2.     Type in the duration of simulation to be the maximum number of years for the simulation to run.

3.     Type in the threshold of similarity to be the value that the Bhattacharya index of population distribution across subsequent years must pass for the simulation to stop. This indicates that the model has reached a quasi-equilibrium (stationary) state. This value must be between 0 and 1. Set this value as 1 if you want the simulation to run the number of years set in duration of simulation.

4.     Type in the value of the Resource Following – Alpha parameter.

5.     Type in the value of the Strength of Sociality – Beta parameter.

6.   Slide the bar to indicate the Proportion reference vs. working memory – Kappa parameter. When all of the actionable memory is from the preceding year. When  the actionable memory is entirely the reference memory.

7.     Type in the value of the Spatial Scale of Sociality – Lambda parameter.

8.     Type in the value of the Diffusion Parameter – Epsilon parameter.

 

On the right side of the screen labeled as Resource:

1.   Slide the bar to indicate the Resource Space Distribution which is the spatial scale of the resource pulse

2.   Slide the bar to indicate the Resource Time Distribution which is the time duration of the resource pulse

3.     Type in the Initial Resource Position in Space value. This value is  which is the spatial coordinate of the resource peak for summer for the first year. The spatial coordinate of the resource peak for winter is

4.     Type in the Initial Resource Position in Time value. This value is  which is the timing of the resource peak for summer for the first year. The spatial coordinate of the resource peak for winter is  In this simulation, we set  as the duration of period i.e. there are 100 days in one year.

5.     Type in the Resource Change in Space value. This value is  which is the rate of change of the peak location. If there is no trend in space in the resource. If there is a shift of the resource towards the extremes of the domain.

6.     Type in the Resource Change in Time value. This value is  which is the rate of change of the peak location. If there is no trend in time in the resource. If there is a shift of the resource towards the extremes of the domain.

7.     Type in the Stochasticity in Space value. This value is  which is standard deviation of the peak location. If there is no stochasticity in space in the resource.

8.     Type in the Stochasticity in Time value. This value is  which is standard deviation of  the peak time. If there is no stochasticity in time in the resource.

9.     Type in the Years of Stable Resource value. This value is the number of years that the resource remains constant before any trend or stochasticity is accounted for in the simulation.

11.  Press View Resource to see visualize the resource. The resource image will display in the middle panel underneath Resource.  

 

Once you have set all the values, you are now ready to run the simulation. Press Run Model to run the simulation. In the bottom-right corner, you will see a message that states “Calculation in Progress. This may take a while…” which indicates that the simulation is running.

 

Once the simulation has completed, one table and three images will be produced in the middle panel of the screen.

Under Migratory Population:

1.     The table displays values for:

a.     FE : the foraging efficiency metric for the last year of the population and the last year of the resource.

b.     avgFE : the average foraging efficiency for the last 20 years of the simulations. If the simulation ran less than 20 years, then all of the years are of the simulation are accounted for in the average calculation.

c.     TE : the total mismatch metric for the last year of the population and the last year of the resource

d.     avgTE : the average total mismatch for the last 20 years of the simulations. If the simulation ran less than 20 years, then all of the years are of the simulation are accounted for in the average calculation.

e.    SA_total : the spatial adaptation index. If  then the SA index will be marked as NA.

f.    TA_total : the temporal adaptation index. If  then the TA index will be marked as NA.

g.     final_similarity : the Bhattacharya index of population distribution across the last two subsequent years

h.     n.runs : the number of years the simulation ran

2.     A plot of the behavior of the population for each year of the simulation with time in the x-axis and space in the y-axis.

Under Memory:

3.      Four plots representing the memory of the population.

a.     Top-left image: The centroid of the annual movement of the population with dark blue indicating year 0 and red representing the last year of the simulation.

b.     Top-right image: The annual foraging efficiency across years

c.     Bottom-left image: Migration timing parameters for each year, with orange segments indicating arrival and departure from the summering (northern) grounds, and the blue segments indicating timing of arrival and departure at the wintering grounds

d.     Bottom-right image: Migration arrival and departure location across years, with blue and orange indicating winter (southern) and summer (northern) locations.

Under Resource:

4.     Plot of the resource for each year. This is the same picture that is generated when you hit View Resource.

 

To save the parameters of a particular run, press Download Results, which will download a .csv files with the model parameters, resource information, and metrics.
